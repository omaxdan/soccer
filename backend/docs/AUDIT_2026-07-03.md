## 5. Matches Page Confidence Score — BUILT THIS AUDIT

No confidence score existed anywhere (verified). Built:

- **Migration 016**: `match_intelligence.confidence_score numeric,
  confidence_band text`
- **Formula** (in `processMatchIntelligencePartial`, backend, zero API calls):
  Pick = readiness-gap side. Each evidence stream is a signed edge toward the
  pick, clamped at a saturation constant, weight-blended:

  | Component | Saturation | Weight |
  |---|---|---|
  | Readiness gap | 30 pts | 30 |
  | Strength gap (own vs own) | 30 | 20 |
  | Injury burden gap (opponent's burden helps) | 40 | 15 |
  | Congestion gap (post-polarity-fix) | 50 | 10 |
  | Travel gap | 1500 km | 10 |
  | Venue advantage gap | 40 | 7 |
  | Squad stability gap | 40 | 5 |
  | Motivation proxy gap | 20 | 3 |

  `confidence = 50 + 50 × Σ(edge×w)/Σw`, missing components renormalize
  (never count as negative evidence), gated on ≥2 genuinely independent
  corroborating streams beyond gap+motivation (else null — no false
  precision). Verified by simulation: full agreement → 93.5 Strong,
  contradiction (strength+injuries oppose a big gap) → 48.1 Avoid, tiny
  neutral gap → 53 Avoid, away-side picks handled, renormalization correct.
- **Bands (per spec)**: ≥95 Elite | 85-94 Strong | 70-84 Moderate |
  55-69 Risky | <55 Avoid
- **Frontend**: CONF % column added after PICK on the matches page,
  color-banded, dash + tooltip when not yet computed.

Note: an *Elite* score requires near-total cross-stream agreement — by
design rare. If the observed distribution skews too conservative once real
data flows, tune saturations, not weights, first.
